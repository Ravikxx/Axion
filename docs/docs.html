Installation

Axion is a Node.js CLI published on npm. You need Node 18 or later.

# Install globally
$ npm install -g axion-cli

# Verify
$ axion --version
Tip: If axion isn't found after install, make sure your npm global bin directory is on your PATH. Run npm bin -g to find it.

Requirements

  • Node.js 18+ — required
  • An API key — for Claude, GPT-4o, Gemini, or Groq. Or use Lumen for free with no key.
  • Python 3 (optional) — needed for computer use / screen overlay
  • Tesseract (optional) — for OCR / find-by-text in computer use

Run axion /doctor after install to check your environment.

First Launch

Run axion to start. If no API key is set, the onboarding wizard walks you through setup.

$ axion

Welcome to Axion!
Pick a model to get started:
› Claude (Anthropic) · GPT-4o (OpenAI) · Gemini · Groq · Lumen (free)

The wizard lets you pick a model and enter your API key. If you select Lumen, no key is needed — it's free and hosted by Axion Labs.

Skipping the wizard

Set a key manually and jump straight in:

# Set your API key
$ axion
› /api claude sk-ant-…

# Or start with the free model
$ axion --model lumen

Code vs Chat Mode

Axion has two modes. You can switch between them at any time.

Code mode (default)

The agent has access to all tools: reading and writing files, running shell commands, searching the web, using the browser, git operations, and more. Use this for actual coding tasks.

$ axion # starts in code mode

Chat mode

No tools — just a conversation. Faster, cheaper, and better for questions, brainstorming, writing, and explanations where you don't need the agent touching files.

$ axion --chat # start in chat mode
# or switch mid-session:
› /chat
Tip: Lumen works best in chat mode. It's a general assistant, not a coding agent — it doesn't have tool access through the CLI.

Memory

After each session, Axion silently extracts key context and injects it into the next session's system prompt.

It remembers things like:

  • Project name and description
  • Decisions you made ("we're using Postgres, not SQLite")
  • File paths and structure
  • Preferences ("always use TypeScript")

Viewing and managing memories

› /memory # view saved memories
› /memory clear # clear all memories

Memories are stored locally in ~/.axion/memories.json — they never leave your machine unless you're using a cloud API.

Discord Bot

Run Axion as a Discord bot so your team can queue tasks from any channel.

# Set your bot token
› /api discord <bot-token>

# Start the bot daemon
$ axion-discord

# Override the model
$ axion-discord --model lumen

Once running, message the bot in Discord. It processes tasks in a queue — one at a time — and streams responses back to the channel. Members can cancel the current task by typing cancel.

Get a bot token from the Discord Developer Portal. Enable "Message Content Intent" under Bot settings.

Chrome Extension

The Chrome extension connects your browser to Axion, letting the agent read pages, click elements, and fill forms.

Setup

  1. Start the Axion web UI: axion /web or axion-serve
  2. Install the extension from the GitHub repo (load unpacked from extension/)
  3. The extension auto-connects to localhost:3000

What it can do

  • Read the full text and DOM of any page
  • Click buttons and links
  • Fill and submit forms
  • Take screenshots
  • Navigate to URLs
Tip: The extension shares your saved API keys from the CLI automatically — no need to reconfigure.

MCP Servers

Axion supports the Model Context Protocol — plug in any MCP server to add new tools and data sources.

First-party servers

  • Blender — control Blender scenes, create objects, run scripts. Start with axion-blender or use the Python script in mcp-servers/blender/.

Adding a custom MCP server

› /mcp add <name> <command>

Any MCP-compatible server works — database connectors, internal tools, third-party APIs. See the MCP spec for building your own.
